# home-ops

GitOps repository for my home Kubernetes cluster.

This repo manages a single Talos Linux node that runs both the Kubernetes control plane and workloads. Flux reconciles the cluster from this repository, with SOPS-encrypted secrets and app manifests organized by namespace.

This repository was originally forked/generated from the onedr0p cluster template:

- https://github.com/onedr0p/cluster-template
- https://github.com/onedr0p/home-ops

The template has been adapted for this cluster and should be treated as an operations repo, not a generic bootstrap template.

## Cluster

Current shape:

```text
Node:        k8s-node-0
Node IP:     192.168.16.100
API VIP:     192.168.16.254
OS:          Talos Linux
CNI:         Cilium
GitOps:      Flux
Secrets:     SOPS + age
Ingress:     Envoy Gateway + Cloudflare Tunnel
Storage:     OpenEBS hostpath
```

## Repository Layout

```text
bootstrap/      Bootstrap-only secrets and setup material
kubernetes/     Flux-managed Kubernetes manifests
talos/          Talos source config, secrets, patches, and generated clusterconfig
JUST_COMMANDS.md  Common operational commands
```

Generated Talos files live under:

```text
talos/clusterconfig/
```

Those generated files contain sensitive material and are ignored by Git.

## Common Commands

List available commands:

```bash
just -l
just talos -l
just kube -l
```

Reconcile Flux with the latest pushed Git state:

```bash
just kube reconcile
```

Check Kubernetes nodes:

```bash
kubectl --kubeconfig /home/dkwise/home-lab/repos/dkw99/home-ops/kubeconfig get nodes -o wide
```

Check Talos health:

```bash
talosctl --talosconfig /home/dkwise/home-lab/repos/dkw99/home-ops/talos/clusterconfig/talosconfig \
  --endpoints 192.168.16.100 \
  --nodes 192.168.16.100 \
  health
```

## GitOps Workflow

Normal workflow:

```bash
git status
git add <files>
git commit -m "<message>"
git push
just kube reconcile
```

Flux pulls from Git. Local uncommitted changes are not applied by Flux.

## Secrets

Secrets are encrypted with SOPS. Do not commit decrypted files or generated credentials.

Ignored local/generated examples:

```text
kubeconfig
talos/clusterconfig/talosconfig
talos/clusterconfig/*.yaml
*.decrypted~*.yaml
```

## Talos Notes

Talos source-of-truth inputs:

```text
talos/talconfig.yaml
talos/talenv.yaml
talos/talsecret.sops.yaml
talos/patches/
```

Regenerate Talos config:

```bash
just talos generate-config
```

Applying Talos config can reboot the only node and temporarily take the cluster down. Review [Just Commands](./JUST_COMMANDS.md) before running disruptive Talos commands.

## Networking Notes

The primary node network is on `192.168.16.0/24`.

Home Assistant also uses Multus for local IoT discovery on a secondary interface:

```text
Interface: ens19
HA IoT IP: 192.168.10.100/24
```

The secondary IoT interface is intended for local device discovery/control only and should not become the default route.
